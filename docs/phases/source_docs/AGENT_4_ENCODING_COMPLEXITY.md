## 1. Scientific Context

### 1.1 Research Question

What is the **exact computational complexity** of the IsalGraph encoding algorithm (graph → string)? This is distinct from Agent 3 (which compares wall-clock times); here we characterize the **theoretical and empirical scaling** of:

1. **Greedy GraphToString** (single starting node): The algorithm described in `src/isalgraph/core/graph_to_string.py`.
2. **Greedy-min** (minimum over all N starting nodes): Runs GraphToString N times.
3. **Canonical string** (exhaustive backtracking): The algorithm in `src/isalgraph/core/canonical.py`.

### 1.2 Theoretical Complexity Analysis

From the codebase and preprint (López-Rubio, 2025):

#### Greedy GraphToString (single start)

The algorithm's main loop runs until all nodes and edges are inserted. At each iteration:
1. Generate displacement pairs sorted by |a| + |b|: There are (2M+1)² pairs where M = current node count. Generation is O(M²); sorting is O(M² log M).
2. For each pair (a, b), walk the CDLL by |a| + |b| steps: O(M) per pair.
3. Check node insertion (V/v) or edge insertion (C/c): O(degree) per check.

The outer loop runs at most |V| + |E| times (each iteration inserts one node or one edge). For each iteration, in the worst case, all O(M²) pairs are tried, each requiring O(M) pointer walks.

**Worst-case per iteration**: O(M² · M) = O(M³) where M = current node count.
**Total worst-case**: Σ_{m=1}^{N} O(m³) = O(N⁴) for node insertion, plus edge insertion costs.

For sparse graphs (|E| = O(N)): The pair search terminates early (small displacement needed), so empirically the constant is much smaller.

For dense graphs (|E| = O(N²)): More edges to insert, each potentially requiring large displacement searches. Empirically closer to O(N⁴).

**Claim from preprint**: López-Rubio (2025), Idea.pdf, states: "My graph-to-string conversion algorithm would have cubic complexity, with N the number of nodes: N strings are computed (one per node), and each string requires O(N²) operations because at most there are N² edges." This refers to greedy-min as O(N · N²) = O(N³). **We must empirically verify whether this is accurate.**

#### Greedy-min (all starts)

Runs GraphToString N times (once per starting node), each with the above complexity. **Total**: O(N) × O(single_run) = O(N⁴) or O(N⁵) depending on the single-run complexity.

#### Canonical (exhaustive)

The canonical search branches over neighbor choices at each V/v step. At each V/v step, the number of candidate neighbors is at most the degree of the primary/secondary pointer's graph node minus already-inserted neighbors. For a graph of maximum degree Δ:

**Worst-case branching**: Δ! at each of N-1 node insertions → Π_{k=1}^{N-1} min(Δ, k) branches.
**For complete graphs (Δ = N-1)**: (N-1)! branches → O(N! · N²) total.
**For trees (Δ bounded)**: Bounded branching → polynomial for fixed Δ.

This is inherently exponential for general graphs but tractable for sparse, bounded-degree graphs.

### 1.3 What We Measure Empirically

1. **Scaling exponent α** from T(n) ≈ c · n^α, estimated via log-log OLS regression.
2. **Density dependence**: How does encoding time scale with edge density ρ = |E| / (N choose 2)?
3. **Family dependence**: Trees vs. cycles vs. random vs. complete — do structural properties affect complexity?
4. **String length scaling**: How does |w| scale with N and |E|?
5. **Greedy vs. canonical**: Quantify the overhead of exhaustive search.

### 1.4 Comparison Targets

Compare empirical encoding complexity against:
- **Exact GED**: NP-hard, empirically exponential (Blumenthal & Gamper, 2020: times out at ~16 nodes).
- **Bipartite GED approximation**: O(n³) via Hungarian algorithm (Riesen & Bunke, 2009).
- **Graph canonicalization**: Practically O(n) for most graphs via Nauty (McKay & Piperno, 2014), exponential worst case.
- **WL subtree kernel**: O(hm) for h iterations (Shervashidze et al., 2011).

### 4.2 Synthetic Graph Generation (`synthetic_generator.py`)

```python
"""Synthetic graph generation for encoding complexity analysis."""

import networkx as nx
import numpy as np

def generate_graph_family(
    family: str,
    n_nodes: int,
    seed: int = 42,
    instance: int = 0,
) -> nx.Graph:
    """Generate a graph from a specified family.
    
    Args:
        family: One of "path", "star", "cycle", "complete",
                "binary_tree", "ba_m1", "ba_m2", "gnp_03",
                "gnp_05", "grid".
        n_nodes: Target number of nodes.
        seed: Random seed.
        instance: Instance index (for families with randomness).
    
    Returns:
        Connected undirected NetworkX graph.
    """
    rng_seed = seed + instance * 1000 + n_nodes
    
    if family == "path":
        return nx.path_graph(n_nodes)
    elif family == "star":
        return nx.star_graph(n_nodes - 1)
    elif family == "cycle":
        return nx.cycle_graph(n_nodes)
    elif family == "complete":
        return nx.complete_graph(n_nodes)
    elif family == "binary_tree":
        # Balanced binary tree with n_nodes
        depth = int(np.ceil(np.log2(n_nodes + 1))) - 1
        G = nx.balanced_tree(r=2, h=depth)
        # Trim to exact size
        while G.number_of_nodes() > n_nodes:
            G.remove_node(max(G.nodes()))
        return G
    elif family == "ba_m1":
        return nx.barabasi_albert_graph(n_nodes, 1, seed=rng_seed)
    elif family == "ba_m2":
        return nx.barabasi_albert_graph(n_nodes, 2, seed=rng_seed)
    elif family == "gnp_03":
        G = nx.gnp_random_graph(n_nodes, 0.3, seed=rng_seed)
        # Ensure connected
        if not nx.is_connected(G):
            G = max(nx.connected_components(G), key=len)
            G = nx.Graph(nx.subgraph(G, G))
            G = nx.convert_node_labels_to_integers(G)
        return G
    elif family == "gnp_05":
        G = nx.gnp_random_graph(n_nodes, 0.5, seed=rng_seed)
        if not nx.is_connected(G):
            G = max(nx.connected_components(G), key=len)
            G = nx.Graph(nx.subgraph(G, G))
            G = nx.convert_node_labels_to_integers(G)
        return G
    elif family == "grid":
        r = int(np.ceil(np.sqrt(n_nodes)))
        c = int(np.ceil(n_nodes / r))
        G = nx.grid_2d_graph(r, c)
        G = nx.convert_node_labels_to_integers(G)
        while G.number_of_nodes() > n_nodes:
            G.remove_node(max(G.nodes()))
        return G
    else:
        raise ValueError(f"Unknown family: {family}")
```

### 4.3 Timing Experiments

#### Experiment 1: Greedy Single-Start Scaling

For each family and size n:
1. Generate graph G.
2. For each starting node v ∈ {0, 1, ..., min(n-1, 4)}:
   - Time `GraphToString(sg).run(initial_node=v)` with 25 reps.

3. Compute median time.


### 4.4 Scaling Regression

```python
def fit_polynomial_scaling(n_values, times):
    """Fit T(n) = c * n^alpha via log-log OLS.
    
    Returns alpha, c, R², and a flag indicating if
    exponential fit is better.
    """
    log_n = np.log(n_values)
    log_t = np.log(times)
    
    # Polynomial fit
    slope, intercept, r_poly, _, _ = stats.linregress(log_n, log_t)
    
    # Exponential fit: T(n) = c * b^n → log(T) = n*log(b) + log(c)
    slope_exp, intercept_exp, r_exp, _, _ = stats.linregress(n_values, log_t)
    
    if r_exp**2 > r_poly**2 + 0.05:  # Exponential clearly better
        return {
            "model": "exponential",
            "base": float(np.exp(slope_exp)),
            "c": float(np.exp(intercept_exp)),
            "R2": float(r_exp**2),
        }
    else:
        return {
            "model": "polynomial",
            "alpha": float(slope),
            "c": float(np.exp(intercept)),
            "R2": float(r_poly**2),
        }
```
