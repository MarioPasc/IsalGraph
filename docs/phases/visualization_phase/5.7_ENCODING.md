## 3. Per-Claim Specification

### C4.1 — Scaling Exponents per Graph Family

**Population** (full width, 2×2 panels): See base document §3.12(A).
- (a) Log-log: greedy single-start time vs N. One line per family (FAMILY_COLORS). Reference dashed lines O(N²), O(N³), O(N⁴).
- (b) Log-log: canonical time vs N. Exponential families in dashed.
- (c) Bar chart: α per family for greedy (blue) and canonical (orange). R² annotations.
- (d) Scatter: observed vs predicted (synthetic → real validation). Identity line. R².

**Individual** (full width, 1×3): Path P₈, BA(8,m=2), K₈. Each: graph drawn + canonical string + encoding time.

**Table**: Per family. Columns: Family, Density, α_greedy, R², α_canonical (or "exp"), R², Model.

### C4.2 — Density Dependence

**Population** (single-column): Heatmap. X = density p ∈ {0.1,...,0.9}. Y = node count N. Color = log₁₀(canonical time). Contour lines at 1s, 10s, 60s, 300s. Cividis colormap. This is the "feasibility map."

**Individual** (single-column, 1×2): G(10,0.2) vs G(10,0.8). Graphs drawn + times annotated. "10× density increase → XXx time increase."

**Table**: Grid of N × p with times. "TO" for timeout. Best/worst cells highlighted.

### C4.3 — Greedy vs Canonical Overhead

**Population** (single-column): Bar chart of ratio T_canonical / T_greedy per family. Log y-axis. Color by density class.

**Individual** (full width, 1×3): One graph (e.g., BA(8,m=2)): (a) graph, (b) table of greedy strings per starting node with best highlighted, (c) canonical string. Alignment if they differ. Times annotated.

**Table**: Per family. Columns: T_greedy_min, T_canonical, Overhead ratio, String match %.
