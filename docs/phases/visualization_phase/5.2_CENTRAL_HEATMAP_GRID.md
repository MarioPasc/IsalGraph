## 2. Data Needed

From Agent 0:
- `data/eval/ged_matrices/{dataset}.npz` → `ged_matrix` key.
- `data/eval/levenshtein_matrices/{dataset}_exhaustive.npz` → canonical Levenshtein.
- `data/eval/levenshtein_matrices/{dataset}_greedy.npz` → greedy-min Levenshtein.

**Missing data — greedy single-start**: Agent 0 does NOT precompute a separate Levenshtein matrix for greedy single-start (only greedy-min and exhaustive). To fill row 1 of the grid, the agent must:

1. Load `data/eval/canonical_strings/{dataset}_greedy.json` — this stores the greedy-min result but may include per-starting-node data.
2. If per-starting-node strings are available: extract the `initial_node=0` string for each graph and compute pairwise Levenshtein from those.
3. If not: for each graph, run `GraphToString(sg).run(initial_node=0)` to get the single-start string, then compute pairwise Levenshtein. This is fast (seconds per dataset).

Use `python-Levenshtein` (C-accelerated) for the pairwise computation.

## 3. Implementation Plan

### 3.1 The 3×5 Grid Figure

See addendum §A.4 for full spec. Key points:

- **3 rows**: greedy_single, greedy_min, exhaustive (canonical).
- **5 columns**: iam_letter_low, iam_letter_med, iam_letter_high, linux, aids.
- **Each cell**: hexbin density plot with GED on x-axis, Levenshtein on y-axis.
- **Identity line** (gray dashed), **OLS regression line** (Paul Tol red).
- **Annotation**: Spearman ρ and Pearson r in top-left of each cell.
- **Shared colorbar** on the right.
- **Two versions**: raw scales and z-normalized.
- Use `matplotlib.colors.LogNorm()` for hexbin color mapping.
- Use `sharex='col', sharey='row'` for aligned axes.
- Bold label on greedy-min row to flag it as the recommended practical method.
- Full page: `figsize=(7.0, 8.0)`.

### 3.2 The Companion Table

15 rows (3 methods × 5 datasets). Columns: Dataset, Method, Spearman ρ [95% CI], Pearson r [95% CI], Lin's CCC, OLS slope, n_pairs.

If bootstrap CI data is available from Agent 1's stats JSON, use it. If not, compute bootstrap CI here (1000 resamples is sufficient for a table — this is fast on the flattened distance vectors).

Highlight `\best{}` = highest ρ per dataset (across methods), `\worst{}` = lowest.

Use `generate_dual_table()` from infrastructure.
