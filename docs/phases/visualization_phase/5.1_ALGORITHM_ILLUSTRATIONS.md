## 3. Implementation Plan

### 3.1 State Extraction — S2G (`generate_s2g_walkthrough`)

Run StringToGraph on progressively longer prefixes of the chosen string. For prefix `w[:k]`, create a fresh `StringToGraph(w[:k], directed=False)`, call `.run()`, then extract:
- The output SparseGraph → convert to NetworkX via adapter.
- CDLL state: `s2g._cdll` (inspect internals for `_next`, `_prev`, `_data`, `_size`).
- Pointer positions: `s2g._primary_ptr`, `s2g._secondary_ptr`.
- Compute what changed between step k−1 and step k (new node? new edge? pointer moved?).

**CRITICAL**: Before committing to an example string, run it through `StringToGraph` and verify the output graph is non-trivial (4–5 nodes, at least 1 cycle). Good candidates:
- The canonical string of K₄ (complete graph on 4 nodes).
- The canonical string of the "house graph" (square + triangle on top).
- A manually crafted string like `"VNVnCV"` — but VERIFY it produces a valid connected graph.

**Approach**: First run `canonical_string()` on a known small graph (e.g., `nx.cycle_graph(4)`) to get a verified string, then use that string for the walkthrough.

### 3.2 State Extraction — G2S (`generate_g2s_walkthrough`)

This is harder because GraphToString's main loop is monolithic. Two strategies:

**Strategy A (recommended)**: Add a `run_traced()` generator method to `GraphToString` that `yield`s state after each structural instruction (V/v/C/c). This is clean but modifies core code — check with the project lead.

**Strategy B (conservative)**: Instrument via prefix reconstruction. For each prefix of the output string w produced by `G2S(G, v₀)`, run `S2G(w[:k])` and record the graph state. This doesn't give you the displacement search information but gives you the graph/CDLL evolution. Annotate displacements from the string itself (count N/P/n/p before each structural instruction).

**Strategy C (pragmatic)**: Run G2S once to get the full string. Then parse the string into segments (each segment = pointer movements + one structural instruction). For each segment, run S2G on the cumulative prefix to get the state.

def test_roundtrip_consistency():
    """The graph from S2G walkthrough must equal the input to G2S walkthrough."""
    w = "chosen_string"
    s2g = StringToGraph(w, directed=False)
    sg_s2g, _ = s2g.run()
    
    g2s = GraphToString(sg_s2g)
    w_prime, _ = g2s.run(initial_node=0)
    
    s2g2 = StringToGraph(w_prime, directed=False)
    sg_g2s, _ = s2g2.run()
