## 2. Illustrative Figures

These figures are NOT tied to a specific hypothesis — they build visual intuition.

### `fig_example_pairs.pdf`

Shared with H1.1 individual figure (5.3). If 5.3 has already generated `individual_concordant_pairs.pdf`, this figure can be a copy or a reformatted version specifically styled for the paper's "graphical examples" section. See base document §3.15.

### `fig_string_alignment.pdf`

Full width, 2 rows. Large-format color-coded alignment strips: Row 1 = similar pair, Row 2 = dissimilar pair. Each character is a colored cell: match (green), substitute (red), insert (blue), delete (yellow). Two-row format (source above, target below). Use `draw_alignment()` from infrastructure. Legend below.

### `fig_heatmap_comparison.pdf`

Full width, 1×3 panels. For ~60 graphs from IAM Letter LOW (7 per class × 9 classes, randomly sampled with seed=42):
1. GED distance matrix (N×N image).
2. Levenshtein (exhaustive) matrix.
3. Levenshtein (greedy) matrix.
All three: same row/column ordering (sorted by class, then within-class by GED centroid distance), same viridis colorscale (after min-max normalization), thin black lines at class boundaries.

### `fig_nearest_neighbors.pdf`

Full width, 2 rows. Each row: center = query graph drawn. Top 5 = 5-NN by GED. Bottom 5 = 5-NN by Levenshtein. Shared neighbors outlined with green border. Annotate "Overlap = X/5". Select query graphs with `select_knn_example()`.

### `fig_class_clustering.pdf`

Full width, 1×2 panels. Left = 2D MDS(Levenshtein, exhaustive) for IAM LOW, colored by class. Right = 2D MDS(GED). Larger point size than H2.1 individual. Class name labels at cluster centroids. Convex hulls per class (light shading). This is styled specifically for the "topological similarity preservation" section.

## 3. CLI Orchestrator (`eval_visualization.py`)

```python
"""CLI entry point for paper figure generation.

Dispatches to sub-agent modules based on --hypotheses and --artifacts flags.
Can run all figures or a subset.
"""

# CLI flags (see base document §5.2):
# --data-root, --results-root, --output-dir, --seed
# --hypotheses: all | H1_1,H1_2,... | central_heatmap | algorithm
# --artifacts: all | population | individual | table
# --datasets: all | iam_letter_low,...
# --dpi: 300
```

The orchestrator imports from each sub-agent's module and calls the appropriate generation functions. It collects `SelectedExample` instances from all sub-agents and writes the combined `selected_examples.json`. It generates `figure_manifest.json` by scanning `paper_figures/` for all PDF/PNG/TEX files.

### `figure_registry.py`

Maps hypothesis IDs to module paths (see base document §5.1). Used by the CLI to dispatch.
